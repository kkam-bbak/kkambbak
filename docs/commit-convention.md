## 📌 Commit Message Convention

커밋 메시지는 다음 형식을 따릅니다:

```
<타입>: [요구사항 ID] 작업내용 요약
```

#### ✅ 예시
```
Feat: [REQ-0001] 로그인 API 구현
Fix: [REQ-0003] 채점 알고리즘 null 예외 처리
Update: [REQ-0005] PayPal Webhook 이벤트 핸들러 추가
Delete: [REQ-0007] 사용하지 않는 유틸 함수 제거
Refactor: [REQ-0008] FFmpeg 처리 로직 개선
Docs: [Docs] 요구사항 명세서 업데이트
```

---

### ✏️ 타입 (Type)

| 타입       | 설명                                                  |
|------------|-------------------------------------------------------|
| `Feat`      | 새 기능, 파일, 모듈을 추가할 때                      |
| `Update`   | 기존 코드에 기능 추가 또는 설정 변경 시              |
| `Fix`      | 버그 수정                                             |
| `Modify`   | 코드 기능 일부 수정 (버그가 아닌 일반 수정)         |
| `Delete`   | 코드, 파일, 모듈 삭제 시                              |
| `Refactor` | 리팩토링 (동작은 그대로, 코드 구조 개선)             |
| `Docs`     | 문서 수정 (README, 계획서 등)                         |
| `Test`     | 테스트 코드 추가 또는 수정                            |
| `Chore`    | 빌드, 패키지, 설정 파일 등 기타 변경사항             |

---

### 🗂 요구사항 ID 또는 카테고리 (Category)

기본적으로 아래 요구사항 ID 중 작업 내용에 맞는 것을 선택하여 사용합니다.
단, 여러 요구사항에 걸친 공통 작업이나 특정 ID로 분류하기 어려운 경우에 한해 아래 카테고리를 사용할 수 있습니다.

####  요구사항 ID
- `REQ-0001`: 스플래쉬 화면
- `REQ-0002`: 로그인 기능
- `REQ-0003`: 회원가입 프로세스
- `REQ-0004`: 한국어 이름 생성 및 프로필 설정
- `REQ-0005`: 메인 페이지 UI/UX
- `REQ-0006`: 프로필 관리
- `REQ-0007`: 학습 개인화 알고리즘
- `REQ-0008`: 학습 목록 페이지
- `REQ-0009`: 학습 진행 페이지
- `REQ-0010`: 학습 완료 및 결과 페이지
- `REQ-0011`: 롤플레이 목록 페이지
- `REQ-0012`: AI 롤플레이 기능
- `REQ-0013`: 한국어 스피드 퀴즈 게임
- `REQ-0014`: 스피드 퀴즈 결과 페이지
- `REQ-0015`: 나의 학습 내역
- `REQ-0016`: VOD 학습 세션
- `REQ-0017`: 결제 시스템

####  카테고리 (Category)
- `Docs`: 문서 관련
- `Auth`: 로그인, 회원가입, 인증, JWT 관련
- `User`: 사용자 정보, 프로필 관련
- `Payment`: 결제, 구독, PayPal 관련
- `Learning`: 학습 시스템, 채점, 통계 관련
- `Session`: 학습 세션, 문제 관련
- `Video`: 동영상 업로드, 번역 관련
- `File`: 파일 저장, 관리 관련
- `AI`: AI API 연동 (STT, Translation, 이미지 생성)
- `KoreanName`: 한국어 이름 생성 관련
- `Security`: Spring Security 설정 관련
- `UI`: 프론트 UI 관련
- `Common`: 공통 유틸, 인터셉터 등

---

### 💡 팁
- 메시지는 **명확하고 간결하게** 작성합니다.
- 여러 변경이 있을 경우 **여러 커밋**으로 나누는 것이 좋습니다.
- PR 제목과도 일관성을 유지하세요.
